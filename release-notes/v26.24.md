# v26.24

- Added a `te-cli` skill (tabular-editor plugin) for the cross-platform Tabular Editor CLI (`te`): grounded modeling guidance, a TE2-to-TE3 migration reference, and a vendor-sync script that pulls the upstream skill on merge
- Added a consolidated `semantic-model` skill (semantic-models plugin) spanning the design / build / refresh / review lifecycle and routing every change through a `te`-first cascade (`te` CLI, then a model MCP or TOM via `connect-pbid`, then TMDL via the `tmdl` skill); it replaces `review-semantic-model`, which was removed
- Power BI Desktop Bridge integration across the report and model skills. The bridge is Desktop's per-process JSON-RPC named pipe that reloads the report canvas and captures page screenshots. The `pbir-cli` skill drives it through the new `pbir desktop list/refresh/screenshot` verbs to close the loop locally (mutate, refresh, screenshot, inspect) with auto-refresh on save, absolute report paths, and live local-engine query routing for thick reports (new `references/desktop-integration.md`); `connect-pbid` pairs it with TOM to verify model edits against the rendered report, drives the pipe raw when `pbir` is absent, and documents port-to-report correlation via the msmdsrv process tree (new `references/desktop-bridge.md`); `pbir-format` covers the `powerbi-desktop` npm CLI for the same loop. Reload semantics verified empirically: `file.reload` picks up on-disk PBIR edits, but theme files and TMDL need a close and reopen
- `connect-pbid` script fixes: `create-field-parameter.ps1` built a non-functional field parameter through two nonexistent TOM APIs while reporting success (now uses `GroupByColumns` + `JsonExtendedProperty`, fails fast, and cleans up); `debug-dax.ps1` polls for async DAXEvaluationLog delivery instead of missing events; `daxlib.sh` errors clearly when `jq` is missing and supports `--help`; the quickstart dedupes ports and skips thin-report engines; `load-tmdl.ps1` warns when `-AddMeasure` runs without `-Save`
- pbi-desktop hooks: `refresh-cache` and `check-ri` now read executed `.ps1` contents (they never fired for the recommended `-File` workflow); the validate hooks no longer block the plugin's own bundled scripts; hook README synced to `hooks.json`. The pbip PBIR hook validates page `displayOption` against the schema enum and matches nested and absolute `.Report` paths
- `pbir-format`: repaired the bundled K201 example (phantom `pages.json` entries, dangling theme reference, stale bookmark sections and queryRef) in both plugin copies; resolved the `midColor` token contradiction (use `neutral`); refreshed schema-version drift notes
- Expanded the report skills (`review-report`, `pbi-report-design`, `create-pbi-report`, `deneb-visuals`, `svg-visuals`, `python-visuals`, `r-visuals`, `pbir-format`) with reference material on accessibility, performance, interactivity, chart selection, mobile layout, filter panes, theming, and PBIR shapes
- Refined the `dax` skill's performance-optimization, patterns, engine-internals, and model-optimization references (contributed by Justin Martin)
- Rebuilt the `tmdl-validate` binaries (v0.2.0) for linux-x64 and windows-x64; clarified TMDL indentation spec vs PBIP convention in the tmdl docs
- Added a release-age-bypass hook plus non-personal settings and statusline updates to `useful-stuff`
