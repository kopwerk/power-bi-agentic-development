# v26.20

## Versioning switchover

Starting with this release the marketplace uses **YY.WW** versions (ISO year + ISO week). The previous `0.X.Y` line ends at `0.26.1`; the next release after that would have been `0.26.2`, and is published here as `26.20`. Hotfixes inside a week append `.N`, for example `26.20.1`. See `.claude/rules/releases.md` for the rules; `.claude/scripts/bump-version.py` and `.claude/commands/release.md` updated to match.

## Marketplace layout cleanup

- Removed `.claude-plugin/plugin.json`. Anthropic's documented marketplace layout treats the root as a catalog, not a single distributable plugin. Per-plugin manifests still live at `plugins/<name>/.claude-plugin/plugin.json`; the root keeps only `marketplace.json`.
- **Migration step for existing installs:** run `/plugin marketplace update power-bi-agentic-development` once after upgrading so Claude Code re-reads the manifest. Without that, the cache may briefly show stale metadata.

## Copilot CLI support

- New README section with the documented install paths: `copilot plugin marketplace add` plus per-child-plugin install, the `OWNER/REPO:plugins/<name>` subdirectory shortcut, and the in-session `/plugin install` equivalent.
- Renamed all 8 plugin agents from `*.md` to `*.agent.md` to match Copilot CLI's required convention. Claude Code matches any `*.md` in `agents/`, so the dual extension works in both tools. Verified locally.
- README compatibility note: hooks require **Copilot CLI >= 1.0.26** for `${CLAUDE_PLUGIN_ROOT}` resolution; older builds caused the `bash` invocations in `hooks.json` to resolve to broken paths. The CI workflow now also fires on `releases/**` branches and PRs.
- Added [`useful-stuff/enable-windows-longpaths.ps1`](../useful-stuff/enable-windows-longpaths.ps1) for the Windows pre-flight. Toggles `HKLM:\...\FileSystem\LongPathsEnabled` and `git config --system core.longpaths`; idempotent, self-checks for admin. Required on Windows because this repo's TMDL paths exceed 260 chars.

## pbip-validator: M-expression vs table name collision (new ERROR rule)

`validate_pbip.py` now detects when an M expression in `definition/expressions.tmdl` shares a name with a table in `definition/tables/*.tmdl`. Power BI Desktop puts both in the same member namespace, so a collision fails the model load with `Microsoft.Data.Mashup.Preview; This document contains a duplicate member '<name>'`. Severity is ERROR; suggested fix is to rename the M expression (typical pattern: append ` Query`) and reference it from partitions via `source = #"<Name> Query"`. The `tmdl` skill and the `pbip-validator` agent both document the rule. Reported by a user; the bundled Rust `tmdl-validate` binary in `plugins/pbip/hooks/bin/` does not yet ship this check (the binary's source lives in an out-of-tree `tools/tmdl-validate/` workspace).

## Known limitations

- The longpath PowerShell script has been static-tested only; smoke-test on a Windows machine before relying on it for a production install.
- DataHub V2 API remains undocumented Microsoft internal surface area. The `search_across_workspaces.py` script is fit for governance and audit work; do not depend on it for production automation.
